Я хочу сделать телеграм бота для отслеживания калорий, интегрированного с API OpenAI, чтобы от меня требовалось меньше действий и при этом получался максимально детальный отчет. Бот должен также учитывать бжу баланс, различные витамины и минералы, клетчатку и всё что только можно. При этом через бота можно было бы получить детальный отчет о прогрессе. Также он должен записывать мой вес и показатели тела как только я замеряю их все с помощью Mi умных весов. А также показатели активности из Samsung Health. Возможно через интеграцию с Google Fit или другими агрегаторами. 

# Сценарии использования примерно такие:

## Внесение приема пищи

- Я описываю свой прием пищи и/или прикладываю фото (возможно несколько) также в будущем будет запись голосовых сообщений с расшифровкой с помощью API OpenAI
- Отправляю это боту через телеграм чат. Указываю дату и время, если не указываю, значит они равны дате и времени сообщения.
- Далее серверная часть отправляет это всё в чатгпт и просит вернуть JSON в нужном формате с описанием всех блюд и их пищевой ценности (максимально детально). При этом если информации не достаточно, то он должен предположить
    - Также предусмотри возможность добавления общего описания моей диеты и предпочтений. Если оно заполнено, то его также нужно добавить при обращении к чатгпт
- Далее полученные данные записываются в базу данных на сервере

## Обновление описания моей диеты и предпочтений:

- Я выбираю обновить описание диеты и предпочтений
- Мне возвращается текущее описание
- Я отправляю новое текстом (или голосовым сообщением - в будущем)
- Если отправлено голосовое сообщение - оно расшифровывается OpenAI
- Описание сохраняется на сервере

## Просмотр статистики:

- Я выбираю период для просмотра статистики (день, 3 дня, неделя итд)
- Я выбираю детализацию (только кбжу или дополнительные данные)
- Отправляется запрос на сервер
- Возвращается статистика в красивом легко читаемом формате

## Задать норму калорий и бжу:

- Есть опции: ввести вручную или запросить рекомендацию у бота
- Если введено вручную, текстом, то просто обновляются данные (придумать формат, чтобы можно было легко обновить текстовым сообщением)
- Если введено запросить рекомендацию у бота, то:
    - Собираются данные о моем весе, показателях организма, активности и потребляемых кжбу за последний месяц
    - Отправляются в API OpenAI с просьбой подготовить JSON с рекомендуемыми кжбу и также текстовым пояснением к рекомендации
    - Рекомендация сохраняется в базе данных
    - Бот возвращает рекомендацию
    - Я также вручную ввожу норму, либо ту которую задал бот либо

## Выполнить анализ показателей

- Еженедельно и ежемесячно запускается автоматическая задача анализа прогресса
- Выбираются данные о питании, активности и показателях тела за последнюю неделю/месяц
- Выбираются результаты анализа за:
    - последние 4 недели (для недельных отчетов) и последний отчет за месяц
    - все недели месяца и последние три отчета за месяц
- Отправляются в API OpenAI с просьбой детально прокомментировать текущий прогресс с точки зрения диетологии. Попросить не использовать маркдаун.
- Ответ сохраняется в базу данных с полями:
    - дата
    - тип (ежемесячный/еженедельный отчет)
    - ответ от OpenAI
- Отчет отправляется ботом в личные сообщения

# Получение результатов анализа

- Я указываю за какой период анализ меня интересует (выбираю неделю или месяц)
- Бот возвращает текст анализа

# Технические требования

Решение должно работать на облачных технологиях

Решение должно работать для двух разных аккаунтов в телеграме, а может даже больше в будущем

Решение не планируется масштабировать для коммерческого/массового использования

Решение должно быть легко переносимым на другой сервер

Решение должно быть легко дорабатываемым

**Core Stack:**

1. **Backend:** Python + FastAPI
    - SQLAlchemy как ORM
    - Pydantic для валидации данных
    - python-jose для JWT авторизации
    - python-multipart для работы с файлами
2. **База данных:**
    - PostgreSQL
    - Alembic для миграций
    - psycopg2-binary как драйвер
3. **Очереди и кэширование:**
    - Redis
    - Celery для фоновых задач
    - celery-beat для периодических задач
4. **Контейнеризация:**
    - Docker
    - Docker Compose

**Для хостинга** я бы порекомендовала DigitalOcean:

- Простой и понятный интерфейс
- Предсказуемое ценообразование
- Надежность
- Хорошая документация на русском языке
- Managed Databases для PostgreSQL и Redis

Для такого проекта будет достаточно:

- Droplet: Basic Plan ($6-12/мес)
- Managed PostgreSQL: Starter Plan ($15/мес)
- Managed Redis: Basic Plan ($8/мес)
